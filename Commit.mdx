# Commit Convention (공통)

커밋 메세지 구조는 아래의 구조를 따른다.<br/>
커밋 메세지 각 파트는 빈 줄을 두어 구분한다.

## 구조

> [타입] (옵션): [제목]<br></br> > [본문]

## 예시

> “feat: 로그인 기능 구현”
>
> - Next Auth 패키지 설치
> - Prisma DB 어댑터 추가

## 커밋 타입

타입은 모두 소문자로 한다.

- feat : 새로운 기능 추가
- fix : 버그 수정
- docs : 문서 수정
- style : 코드 포맷팅
- refactor : 코드 리팩토링
- test : 테스트 코드
- chore : 빌드 업무
